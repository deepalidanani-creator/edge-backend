Round 2 — Employee CSV Upload

Technical Design · Implemented

L&D admin uploads a CSV. The API validates the file and each row, compares against tenant data, and returns a preview of proposed changes. Nothing is written to the database.

At a glance

ItemDetail
EndpointPOST /v1/upload/emp-with-cohort-assignment
HeadersX-Tenant-Id · X-User-Role: LND_ADMIN · Idempotency-Key1
File.csv, max 10 MB, multipart field file
All rows valid200 OK + full response
Some rows invalid400 + full response (every row still listed)
File rejected403 / 400 + { "error": "..." }

1 Header required; not stored or replayed in current code.

CSV columns: employee_id, email, name, optional role, optional cohort_names (semicolon-separated display names), optional start_date (parsed, not used in reconcile).

Validation

Whole file fails — wrong role (403); empty, bad extension, too large, parse error (400).

Row errorMeaning
REQUIRED_FIELD_MISSINGMissing id, email, or name
DUPLICATE_EMPLOYEESame employee id twice in file
DUPLICATE_EMAILSame email twice in file
UNKNOWN_COHORTCohort name not in tenant catalog
ORPHAN_COHORT_MEMBERSHIPDB cohort id missing from catalog
EMAIL_CONFLICTEmail change blocked for existing employee

Reconcile outcomes

  • CSV cohort list = desired membership (replace mode).
  • Email change on existing employee = error.
  • valid: true only when every row is valid.

Design questions

TopicAnswer (today)
OutcomePreview JSON only — no DB writes
ProcessingSynchronous; parse → 2 DB reads → in-memory reconcile
Partial failureRow-level; one bad row does not hide others
Repeat uploadIdempotency header required; not replayed yet
Scale (~1.5k rows)Load employees by CSV ids; reconcile in memory
Cohort namesExact match to tenant catalog
Cohort listReplace mode — CSV is source of truth
EmailNo change via CSV; globally unique in DB

Out of scope & trade-offs

  • Writes: EmployeeWriteRepository is implemented (create, update, cohort add/remove). Confirm/apply from preview is not wired — orchestrator + staging are the remaining step.
  • Still open: idempotency replay, mapper logic, start_date in reconcile.
  • Trade-offs: extension-only .csv check · 10 MB in code · no email format validator · preview is read-only.
